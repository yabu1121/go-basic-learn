# ドリル01: ヘルスチェックAPI

## 🎯 目標
Ginを使って最もシンプルなAPIエンドポイントを作成します。

## 📝 要件

以下の仕様でヘルスチェックAPIを実装してください:

### エンドポイント
- **メソッド**: GET
- **パス**: `/health`
- **レスポンス**: 
  ```json
  {
    "status": "ok",
    "message": "API is running"
  }
  ```

### 追加要件
1. サーバーはポート8080で起動すること
2. Ginのデフォルトミドルウェア（Logger, Recovery）を使用すること

## 🧪 テスト方法

```bash
# サーバー起動
go run main.go

# 別のターミナルで実行
curl http://localhost:8080/health
```

### 期待される出力
```json
{
  "status": "ok",
  "message": "API is running"
}
```

## 💡 学習ポイント

- Ginの基本的な使い方
- ルーティングの設定方法
- JSONレスポンスの返し方
- `gin.H` の使い方

## 🚀 チャレンジ課題

余裕があれば以下も実装してみましょう:

1. `/` パスでウェルカムメッセージを返す
2. 現在時刻をレスポンスに含める
3. バージョン情報を返す `/version` エンドポイントを追加
